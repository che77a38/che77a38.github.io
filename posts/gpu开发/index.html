CPU与GPU架构差异

处理器 设计目标 核心数量 缓存结构 并行模式 适合任务
CPU 低延迟、复杂逻辑 4-32 核 多层大容量缓存 串行执行 + 有限并行 通用计算、控制流、分支判断
GPU 高吞吐量、数据并行 数千个流处理器 高带宽小容量缓存 SIMT (单指令多线程) 图形渲染、矩阵运算、视频编码

GPU 专为同时处理海量相似数据而设计，通过将计算任务拆分为大量细粒度子任务，分配给数千个流处理器并行执行，大幅提升吞吐量

GPU 加速的实现机制

  1. 任务卸载：将适合并行处理的计算任务从 CPU 转移到 GPU 执行，释放 CPU 资源NVIDIA 开发者
  2. 数据本地化：将待处理数据从系统内存复制到 GPU 显存，利用高带宽显存总线提升数据访问速度
  3. 并行计算模型：
    • SIMT (单指令多线程)：同一指令控制多个线程处理不同数据块
    • Warp (线程束)：32 个线程为一组，由 GPU 调度器统一调度执行
  4. 专用硬件加速：GPU 集成专用功能单元（如 NVENC 编码单元、张量核心），处理特定任务更高效NVIDIA 开发者
