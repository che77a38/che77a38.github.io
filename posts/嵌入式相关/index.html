嵌入式相关知识点

嵌入式系统：以应用为中心、软硬件可裁剪的专用计算机系统，嵌入到设备中执行特定任务（如智能手环、工业控制器）

特点：

  • 实时性：工业控制需毫秒级响应（如机器人紧急制动）
  • 低功耗：电池设备需优化能耗（如物联网传感器）
  • 资源受限：内存/处理器性能远低于PC（ESP8266仅80KB RAM）

基础知识

电路基础

基尔霍夫定律

  • 基尔霍夫电流定律（KCL）：在集总参数电路中，任意时刻，对任意节点流出或流入该节点电流的代数和等于零

    $$ i_1+i_2+i_3=i_4+i_5 $$

  • 基尔霍夫电压定律（KVL）：在集总参数电路中，任一时刻，沿任一闭合路径绕行，各支路电压的代数和等于零

    $$ U_{S1}-U_{S2}-U_2-U_1=0 $$

模拟数字电路

  • 数字电路是处理数字信号的电路。
    数字信号：数字信号是离散的电信号。只有0和1区分。
  • 模拟电路是处理模拟信号的电路。
    模拟信号：模拟信号是连续的电信号。以电压为纵坐标，时间t为横坐标为例，函数曲线是连续的，不存在突然阶跃的情况

计算机的最小组成

计算机的组成非常复杂,但其基本单元非常简单

  1. 笔记本电路板上有很多芯片,一个芯片就是一个系统
  2. 一个系统由很多模块组成,如加法器,乘法器等

硬件/软件/固件

  • 硬件是物理组件
  • 软件是在系统上运行的可执行代码和数据的集合
  • 固件是介于硬件和软件之间的一类软件,是存储在非易失性存储器中的代码和数据,用于控制硬件设备的操作和功能,如初始化硬件,加载操作系统或直接执行控制功能

存储器

  • ROM(只读存储器)

    只能读取,在出厂前写定内容,无法修改

    用于程序启动和内核功能调用

  • SRAM(静态随机存储器)

    读写速度较快,断电丢失

    用于存储运行时临时性数据

  • Flash(闪存)

    读写速度适中,断电不丢失

    存储程序代码,存储数据,存储固件

I/O

输入/输出 = 交互

共计3种交互

  • 单片机与外部交互
  • 单片机中芯片与芯片交互
  • 单片机与单片机交互

  • 模拟量 Analog

    连续变化的信号,可以在一定范围内取任意数值,且可以无限细分,如语音信号,温度,速度,电压变化

  • 数字量 Digital

    离散的信号, 数字量只能取有限的,特定的数值,通常使用二进制系统(0和1)来表示

模拟量 –采样量化–> 数字量

单片机与外部交互

通常有四种方式

  • GPIO 通用输入/输出 允许输入或输出数字量

  • PWM 脉冲宽度调制 用于产生模拟信号,如控制电机速度,调光LED等;它通过输出快速切换的数字量高低电平,会被电机平均为比较平滑的模拟信号

  • ADC 模/数转换器 它将模拟量转换为数字量,用于从传感器中读取模拟数据

    其作用是将 模拟量 –采样量化–> 数字量

  • DAC 数/模转换器 它将数字量转换为模拟量,用于向执行器生成模拟输出,比如播放音频

    其作用是将 数字量 –逆量化滤波–> 模拟量

  • SPI 串行外设接口

计算机包含五大部分，即运算器、控制器、存储器、输入设备和输出设备。

  • 运算器：负责“算”。所有加减乘除、比较大小、逻辑判断都靠它。
  • 控制器：负责“指挥”。它负责每一步该干嘛、谁先谁后。 。
  • 存储器：负责“记”。记住程序、记住数据、记住中间结果。
  • 输入设备：负责“喂数据”。把你想告诉计算机的内容输入进去。
  • 输出设备：负责“显示结果”。把计算机的答案展示出来。

单片机

单片机又称单片微控制器，它不是完成某一个逻辑功能的芯片，而是把一个计算机系统集成到一个芯片上。相当于一个微型的计算机，和计算机相比，单片机只缺少了I/0设备。概括的讲：一块芯片就成了一台计算机。它的体积小、质量轻、价格便宜、为学习、应用和开发提供了便利条件。单片机的使用领域已十分广泛，如智能仪表、实时工控、通讯设备、导航系统、家用电器等

单片机诞生于1971年，经历了SCM、MCU、SoC三大阶段(集成的东西越来越多,性能越来越强)，早期的SCM单片机都是8位或4位的。其中最成功的是intel的8051，此后在8051上发展出了MCS51系列MCU系统。基于这一系统的单片机系统直到现在还在广泛使用。随着工业控制领域要求的提高，开始出现了16位单片机，但因为性价比不理想并未得到很广泛的应用。90年代后随着消费电子产品大
发展，单片机技术得到了巨大提高，渐渐的32位单片机迅速取代16位单片机的高端地位，并且进入主流市场。而传统的8位单片机的性能也得到了飞速提高，处理能力比起80年代提高了数百倍。高端的32位Soc单片机主频已经超过300Mfz，性能很强，而普通的型号出厂价格跌落至1美元，最高端的型号也只有10美元。当代单片机系统已经不再只在裸机环境下开发和使用，大量专用的嵌入式操作系统被广泛应用在全系列的单片机上。作为掌上电脑和手机核心处理的高端单片机甚至可以直接使用专用的Windows和Linux操作系统

SoC

系统级芯片

System on Chip: SoC 是将整个计算机系统的各个组件（如 CPU、GPU、内存、I/O 接口、通信模块等）集成到单个芯片上的解决方案

典型实例:

  • Raspberry Pi：使用 Broadcom SoC 的单板计算机，适合各种计算和嵌入式应用。
  • 手机芯片（如 Qualcomm Snapdragon、Apple A 系列）：集成了 CPU、GPU、通信模块（如 4G/5G）、图像处理器等。
  • NVIDIA Jetson 系列：用于人工智能和机器学习的 SoC，集成了强大的 GPU 和处理能力。
  • ESP32：虽然它也可以被视为 MCU，但它集成了 Wi-Fi 和蓝牙功能，因此在某些上下文中也可以被视为 SoC。

特点：

  • 功能强大，集成度高
  • 支持多种处理任务和外设
  • 通常具有较高的功耗和性能
  • 适合需要高性能和多功能的应用

MCU

微控制器单元Micro-Controller Unit

定义：MCU 是一种集成电路，通常包括一个或多个处理核心、内存（RAM 和 ROM）、输入输出端口（I/O）和其他外设（如定时器、ADC 等）

用途：MCU 通常用于控制应用，如家电、汽车、工业控制等。它们适合处理简单的控制任务和实时处理

特点

  • 功耗低
  • 处理能力较弱，适合简单任务
  • 通常具有较小的存储空间
  • 设计简单，适合嵌入式系统

典型实例：

  1. Arduino：基于 AVR 或 ARM 微控制器的开发板，广泛用于教育和原型设计。
  2. STM32 系列：STMicroelectronics 提供的 32 位微控制器，广泛应用于嵌入式系统。
  3. PIC 微控制器：Microchip Technology 提供的系列微控制器，常用于工业控制和消费电子产品。
  4. 8051 微控制器：一种经典的 8 位微控制器架构，应用于许多嵌入式系统。
  5. ESP32

DSP

Digital Signal Processor

由德克萨斯仪器公司专门设计同于处理数字信号,因为机器指令对数字信号中的卷积,傅里叶变换,乘法,除法等运算非常快速,常用于: 无线通信,音视频处理,雷达系统等涉及数字信号(音频,图像等)处理的场景

FPGA

Field-Programmable Gate Array

灵活性,可重构性,高实时性

一种可在实际使用前由用户编程的集成电路,利用可编程互连(verilog/VHDL)将可编程逻辑快(大量零散的与,或,非门电路块)按特定功能相连,形成复杂的组合逻辑或时序逻辑

常用于原型设计,加速计算,仪器测控等领域

软件核心层级

  • 裸机开发：直接操作寄存器控制硬件（C语言点灯实验）
  • RTOS（实时操作系统）
    • 必要性: 多任务调度(如同时处理Wi-Fi通信 + 电机控制)
    • 常用系统: FreeRTOS(ESP8266兼容),Zephyr
  • 通信协议
    • 设备间: I²C、SPI、UART（传感器与控制器通信）
    • 网络层: MQTT（物联网云通信）、Modbus（工业控制）

RTOS和裸机开发的区别

裸机开发是嵌入式最基础的前后台轮询架构，RTOS（实时操作系统）是带任务调度的多任务实时架构，二者本质差异在运行机制、调度方式、实时性、代码架构上

对比维度 裸机开发（前后台） RTOS 实时操作系统
核心架构 前后台轮询，单流程串行执行 多任务架构，内核调度、任务切换
任务管理 无任务概念，只有函数调用，执行顺序固定 原生支持多任务、任务优先级、任务状态（就绪 / 阻塞 / 挂起 / 运行）
调度机制 无调度器，纯代码顺序执行 抢占式调度（主流）、时间片轮转，高优先级优先执行
延时实现 空循环软件延时，阻塞整个主循环，全程占用 CPU 系统延时函数，任务进入阻塞态，CPU 切换执行其他任务，资源利用率高
实时性 软实时，响应延迟不确定、抖动大；主循环耗时越长，响应越慢 硬实时，高优先级事件可立即抢占 CPU，响应延迟极小且可预测
中断设计 中断仅做简单标记，复杂逻辑必须丢入主循环；中断与主循环靠全局变量交互 中断只做底层硬件捕获，复杂业务交由任务处理；通过队列 / 信号量实现异步通信
资源竞争 串行执行，无任务竞争；仅中断 + 主循环访问全局变量时存在风险 多任务并发，普遍存在资源竞争，需用互斥锁、临界区、信号量做保护
内存占用 Flash/RAM 开销极小，几 KB Flash、数百字节 RAM 即可运行 内核 + 每个任务独立栈，RAM/Flash 有固定开销，小内存 MCU 无法运行
代码结构 功能复杂后主循环臃肿，逻辑耦合严重（面条代码） 模块化拆分，任务解耦，每个任务只负责单一功能
可维护性 小项目简单；功能增多后迭代、改造成本极高 结构清晰，新增功能直接新建任务，大型项目、团队协作更友好
学习成本 低，只需掌握单片机外设 + C 语言 较高，需额外学习调度、同步互斥、栈管理、死锁排查等 OS 知识
低功耗 休眠时机难控制，功耗优化能力弱 所有任务阻塞时可自动进入深度休眠，低功耗控制更精细

关键差异

  1. 实时性（最核心的差距）
  • 裸机：延迟由主循环周期决定。假设主循环一轮执行耗时 10ms，哪怕外部事件立刻触发，也要等到轮询到对应代码才会处理，紧急事件无法优先响应。中断虽然响应快，但中断里不能写复杂代码，否则会拖慢整个系统。
  • RTOS：工业级 RTOS（FreeRTOS、RT-Thread、uC/OS）均为硬实时。高优先级任务一旦就绪，调度器会立刻暂停低优先级任务，优先执行，响应延迟通常在微秒级，非常适合电机控制、总线通信、精密采样等对响应时间有硬性要求的场景。
  1. CPU 资源利用率
  • 裸机使用for空循环延时：延时期间 CPU 空转，所有功能全部暂停，资源严重浪费。
  • RTOS 使用系统延时：任务延时期间进入阻塞态，CPU 转去执行其他就绪任务，CPU 始终处于高效利用状态。
  1. 代码解耦与项目规模适配
  • 裸机：适合小型简单项目（流水灯、简易按键、单功能传感器）。一旦功能超过 5~8 个，主循环会变得杂乱，新增、删减功能极易引入 bug。
  • RTOS：天生适配中大型复杂项目（智能家居、物联网设备、工业控制器、穿戴设备）。不同业务拆分为独立任务，彼此互不干扰，代码复用、迭代、团队协作效率大幅提升。
  1. 并发安全问题
  • 裸机：串行执行，天然没有多任务竞争问题，仅中断和主循环同时读写全局变量时，简单关中断即可解决。
  • RTOS：多任务同时访问同一硬件 / 全局变量会出现数据错乱、程序跑飞，必须掌握临界区、互斥锁、信号量等同步机制，也是 RTOS 开发的主要难点。
  1. 硬件资源门槛
  • 裸机：8 位单片机、超低配 32 位 MCU（RAM < 1KB）都能正常运行，对硬件无要求。
  • RTOS：内核 + 任务栈会占用 RAM，一般要求 MCU RAM ≥ 2KB 才能流畅运行；8 位低端单片机基本无法搭载 RTOS。

适用场景

🔹 优先选择 裸机开发

  1. 功能极简、逻辑单一的产品：流水灯、简易遥控器、声光报警器、单路传感器；
  2. 硬件资源极度受限：8 位单片机、RAM/Flash 极小的低成本 MCU；
  3. 对实时性无要求，事件响应延迟可容忍；
  4. 极致控成本、大批量量产的低端产品。

补充：复杂裸机项目可搭配状态机优化轮询逻辑，模拟 “伪多任务”，但本质仍是前后台架构，实时性和解耦能力远不如 RTOS。

🔹 优先选择 RTOS 开发

  1. 多业务并行：同时兼顾蓝牙 / WiFi、屏幕显示、电机控制、数据上报的物联网、智能硬件；
  2. 强实时需求：运动控制、伺服电机、CAN 总线、音频采集、精密 AD 采样；
  3. 电池供电低功耗设备：无线传感器、手环、蓝牙模组、NB-IoT 终端；
  4. 大型项目、多人协作、需要长期迭代维护的产品；
  5. 存在大量异步事件（多中断、多通信接口）的场景。

裸机开发也叫大循环轮询系统，是传统单片机最经典的开发模式，分为两部分：

  • 后台（主循环）：上电初始化后，进入 while(1) 死循环，按代码顺序依次执行所有功能函数（轮询）。所有业务串行执行，同一时刻 CPU 只跑一段代码。
  • 前台（中断）：外部紧急事件（按键、串口、定时器等）触发中断，打断主循环，执行中断服务函数（ISR）；中断处理完毕后，回到主循环继续往下执行。

特点：执行顺序写死、串行运行、无任务概念。

RTOS实时操作系统

RTOS 是专门为嵌入式设计的轻量级操作系统，核心是任务调度器：

  1. 把整个程序按业务拆分成多个独立任务（Task），每个任务负责单一功能（如按键任务、显示任务、通信任务）；
  2. 每个任务拥有独立栈空间、独立运行上下文；
  3. 由 RTOS 内核按照优先级、调度算法（抢占式 / 时间片轮转）快速切换任务，宏观上实现 “多任务并发运行”（单核 CPU 微观仍是串行切换）；
  4. 配套信号量、消息队列、互斥锁、事件组等组件，实现任务 / 中断间的同步与通信。

典型逻辑：4 个独立任务，高优先级任务可抢占 CPU。

FreeRTOS架构解析

FreeRTOS 通过精妙的分层架构、确定性调度策略和硬件级优化实现实时低延迟与优先级抢占，其框架设计可通过以下结构图及核心机制解析

FreeRTOS通过 分层微内核+硬件加速调度+中断优先级隔离 实现硬实时

  • 抢占式调度确保高优先级任务即刻响应

  • 优先级继承消除资源竞争延迟

  • FromISR API 缩短中断到任务唤醒路径

  • Tickless模式兼顾低功耗与定时精度

    开发者需合理配置 FreeRTOSConfig.h（如调度策略、堆大小）并优化任务优先级分配，以发挥其极致实时性能

设计哲学：微内核架构仅包含调度、任务同步等必要功能，内核代码精简至几KB，通过 FreeRTOSConfig.h动态裁剪模块（如关闭定时器以节省资源）

任务调度机制（状态转换图）

核心调度策略

  1. 优先级抢占

    • 高优先级任务就绪时立即抢占低优先级任务
    • 优先级位图算法(uxTopReadyPriority)在常数时间内定位最高优先级任务

  2. 时间片轮转

    同优先级任务共享CPU时间(通过SysTick中断触发切换)

  3. 调度确定性保障

    任务切换时间<=1μs（Cortex-M），依赖硬件加速的上下文保存（PendSV异常自动压栈R0-R3等）

低延迟实现机制

中断管理框架

中断优先级分层：

  • 不可屏蔽中断（优先级≤configMAX_SYSCALL_INTERRUPT_PRIORITY）：禁止调用API，响应延迟极低（如电机控制）
  • 可屏蔽中断（优先级＞阈值）：可安全调用API，但可能被内核短暂阻塞
优先级继承解决资源竞争

互斥锁（Mutex）动态提升低优先级任务的优先级，避免优先级反转导致高优先级任务阻塞

低功耗与实时性平衡（Tickless模式）

空闲时关闭SysTick中断，由高精度RTC唤醒并补偿节拍计数，减少功耗且不牺牲调度精度

Arduino与FreeRTOS关系

系统自带N个RTOS任务在后台跑

ESP32 开机后，默认就有一堆系统任务：

  • WiFi 任务
  • 蓝牙任务
  • 系统定时器任务
  • 串口任务
  • 你自己的 loop () 任务

这些任务由 RTOS 调度并发运行，你不用管，RTOS 自动管理。

Arduino用 FreeRTOS 提供多任务、低功耗、WiFi / 蓝牙后台运行等高级能力

setup和loop都是Arduino封装层,底层完全运行在FreeRTOS(ESP32默认的RTOS)之上

简单说: 你写的代码,只是FreeRTOS里的一个任务,RTOS全程都在运行,只是被Arduino隐藏了

ESP32 启动后，真正的执行流程是这样的：

  1. ESP32 芯片上电 → 运行 FreeRTOS 内核（RTOS 已经启动）
  2. FreeRTOS 初始化 → Arduino 核心代码运行
  3. Arduino 核心自动创建一个 FreeRTOS 任务
  4. 这个任务里：先执行 setup()，然后无限循环执行 loop()

因此

  • setup()：任务的初始化部分
  • loop()：任务的循环体
  • 整个程序 = FreeRTOS 中的 1 个任务（Task）

舵机

舵机是一种位置（角度）伺服驱动器。它的核心功能是：接收一个控制信号，并精确地转动到并保持在那个指定的角度位置

当说“舵机”时，通常就是指那种三线、用PWM控制、内部集成了控制电路和电位器的位置伺服电机。它是伺服电机大家族中最常见、最易于使用的一员

伺服电机: 拥有闭环控制的电机

与普通电机的区别：普通直流电机通常只能控制“正转/反转/速度”，而舵机可以控制“转到特定角度”（比如精确转到30度、90度、180度）

与步进电机的区别：舵机内部集成了电机、减速齿轮、控制电路和反馈系统（如电位器），形成了一个闭环控制，所以它通常比开环控制的步进电机精度更高、扭矩更大、更易用。

红色是正极（通常4.8-6V），棕色是地线，橙色是PWM信号线(用于接收控制信号)

  • 切勿将舵机的VCC直接接到Arduino的5V引脚上，尤其是扭矩较大的舵机！Arduino板载的5V稳压器无法提供足够电流，可能导致板子重启或烧毁。
  • 务必使用外部电源（如电池组、独立的5V电源模块）为舵机供电，并将此外部电源的GND与Arduino的GND共地（连接在一起）

使用场景

舵机的“角度保持”特性使其非常适合需要精确定位的场景：

  • 机器人关节：机器人的手臂、腿、头部、手指的转动。
  • 航模/车模：控制飞机舵面（故名“舵机”）、汽车转向。
  • 摄像头云台：控制摄像头上下左右转动，进行追踪。
  • 智能门锁：控制锁舌的弹出和收回。
  • 喂食器/开关：作为机械臂，远程控制一个物理开关的按下或松开

控制原理

舵机的控制需要一种特殊的PWM信号，其特点是：

  • 频率：通常为 50Hz（即周期为20ms）。
  • 脉冲宽度（高电平持续时间）：决定了舵机的角度。
    • 0.5ms 的脉冲宽度 -> 0° 位置
    • 1.5ms 的脉冲宽度 -> 90° 位置
    • 2.5ms 的脉冲宽度 -> 180° 位置

脉冲宽度在0.5ms至2.5ms之间线性变化，舵机的角度就会在0°至180°之间相应变化

像Arduino这样的平台提供了现成的 Servo库，它可以自动帮我们处理这些底层细节

注意事项

  1. 不要手动扭动：在断电时，不要用力强行手动旋转舵机轴，极易损坏内部的齿轮和电位器。
  2. 注意扭矩：选择舵机时要注意其扭矩（kg·cm） 参数，扭矩太小可能带不动你的机械结构。
  3. 供电要充足：务必使用外部电源供电，确保电源能提供足够的电流（单个舵机工作电流可达几百mA，动作时更大）。
  4. 机械限位：很多舵机（如180°舵机）有物理限位，编程时不要试图让它转到超出范围的角度（如myServo.write(200)），否则会听到齿轮“打齿”的噪音，长期会损坏舵机。

电子电路

上拉和下拉电阻

用于确定引脚状态

上拉和下拉电阻的核心目的，是为按键（一个机械开关）在未被按下时，提供一个确定的、稳定的电平状态（高电平或低电平），防止其引脚处于不确定的“浮空”状态，从而避免误触发和电路不稳定

为什么需要他们

原因是浮空状态的危害

当我们把按键连接到微控制器（如 Arduino, STM32, 51单片机）的 GPIO 引脚时，理想情况是：

  • 按键断开 -> 引脚读到一种明确电平（比如 3.3V）
  • 按键闭合 -> 引脚读到另一种明确电平（比如 0V）

但问题在于，当一个引脚什么都不连接（即按键断开时）时，它处于高阻抗状态，也叫“浮空”状态。这个引脚的电平是不确定的，极易受到周围环境（如静电、电磁干扰）的影响，会在高电平和低电平之间随机波动。微控制器会因此读到混乱的信号，导致程序误以为按键被频繁地按下和释放

上拉/下拉电阻就是为了解决这个“浮空”问题而存在的

    • STM32 HAL示例: GPIO_InitStruct.Pull = GPIO_PULLUP;
  2. 如果你的MCU没有内部下拉电阻，而你需要的逻辑又是“高电平有效”，那么就需要外接一个下拉电阻。

电阻值选择：通常选择 4.7kΩ 到 10kΩ 的电阻。阻值太大会无法有效拉平干扰；阻值太小则在按键按下时会产生过大的不必要的电流消耗（费电）,并且可能烧坏端口

PWM引脚

Arduino板上端口数字前带~表示他可以输出pwm信号

PWM是如何产生的？

微控制器（MCU）内部有多个专门的硬件模块，叫做定时器/计数器。它们就像一个个独立的小时钟，可以精确地计数和计时。

  • 频率（Frequency）：由定时器的计数速度决定。定时器计数得多快，一个PWM周期的完成就有多快，从而决定了PWM的频率。
  • 占空比（Duty Cycle）：在同一个定时器内，有专门的比较寄存器。当定时器的计数值小于比较寄存器的值时，输出高电平；大于时，输出低电平。改变比较寄存器的值，就能改变高电平的时间比例，即占空比。

关键点：一个定时器可以同时控制多个输出通道（通常2到4个），为这些通道提供相同的计数基准（即相同的频率），但每个通道都有自己独立的比较寄存器（即可以有不同的占空比）。

为什么同一块板子上的PWM引脚频率会不同？

因为你的开发板（比如常见的Arduino Uno、STM32等）的MCU芯片内部有多个定时器，而不是只有一个。

  • 引脚分组：板子上的PWM引脚并不是平等的，它们被分组连接到不同的定时器上。

    例如，在Arduino Uno（基于ATmega328P）上：

    • 引脚 5 和 6 由Timer0控制，默认频率约 976 Hz。
    • 引脚 9 和 10 由Timer1控制，默认频率约 488 Hz。
    • 引脚 3 和 11 由Timer2控制，默认频率约 488 Hz。

这些定时器在出厂时或被Arduino库初始化为不同的默认值，所以它们产生的默认PWM频率就不同。

  • 你也可以自己改变频率：通过编程，你可以重新配置这些定时器（比如改变它的预分频器设置），从而改变其频率。但请注意：

    • 如果你改变了Timer1的频率，那么所有由Timer1控制的引脚（如Arduino的9和10脚）的频率都会一起改变。
    • 由一个定时器控制的多个引脚，频率必然相同，但占空比可以各自独立设置。

为什么要这样设计？（这样设计的好处）

这种设计主要是为了灵活性和功能分工，主要原因有以下几点：

  1. 兼顾精度与性能：

    • 高频PWM：适合控制开关电源、LED调光（消除人眼可见的闪烁）等需要快速响应的场合。
    • 低频PWM：适合控制舵机（Servo）、生成音频信号等需要精确脉冲宽度的场合。Arduino默认用Timer1（16位定时器）来产生更稳定的低频PWM，就是因为它的精度更高，更适合驱动舵机。

  2. 独立控制，互不干扰：

    • 假设你有一个项目，需要同时：
      • 用高频PWM控制一个电机的转速（比如25kHz）。
      • 用50Hz的PWM信号控制一个舵机。
    • 如果所有PWM引脚都必须是同一个频率，你就无法同时完成这两个任务。
    • 有了多个独立定时器，你就可以将Timer0设置为高频率控制电机，同时将Timer1设置为50Hz来控制舵机，两者互不干扰。

  3. 资源分配与优化：

    • 芯片设计者提供了多个不同位宽（8位、16位）和功能的定时器，以满足各种不同的应用需求。让用户可以根据项目需要，灵活分配这些硬件资源。

总结

  • 不同频率：是因为它们背后连接的是不同的定时器硬件。
  • 同一频率：连接在同一个定时器上的多个PWM引脚，必须有相同的频率。
  • 为什么要这样：为了提供灵活性，允许用户为不同的任务（电机控制、舵机控制、LED调光等）同时设置不同的最佳频率，让各个功能模块互不干扰地工作。

设置的引脚10写入“100”是什么意思？为什么是方波?

在Arduino编程中，使用 analogWrite(pin, value)函数来向标有~的PWM引脚写入值。这个 value的取值范围是 0 到 255。

  • analogWrite(10, 0)：表示占空比为 0%，引脚持续输出低电平（0V）。
  • analogWrite(10, 255)：表示占空比为 100%，引脚持续输出高电平（5V）。
  • analogWrite(10, 100)：表示占空比约为 100/255 ≈ 39.2%。这意味着在一个PWM周期内，引脚只在**39.2%的时间输出高电平（5V），其余60.8%**的时间输出低电平（0V）。

所以，您并没有命令引脚“一直输出高电平”，而是命令它“以大约39%的占空比快速开关”

输出电压 ≈ (100 / 255) × 5V ≈ 0.392 × 5V ≈ 1.96V

这正好完美匹配了您电压表上1.96V的读数！ 电压表告诉你的是：“这个引脚输出的平均电压是1.96V”，而示波器告诉你的是：“这个引脚是通过快速在5V和0V之间切换来产生这个1.96V平均电压的”

PWM引脚的”bug”

以ARDUINO UNO板子为例,会有下面的情况:

  • 舵机影响9和10引脚pwm
  • 蜂鸣器和红外接收影响3和11引脚pwm

原因是: Arduino UNO (基于ATmega328P芯片) 只有3个独立的硬件定时器

Arduino UNO的PWM引脚是这样分配给各个定时器的：

  • Timer0 -> 控制引脚 5 和 6
  • Timer1 -> 控制引脚 9 和 10
  • Timer2 -> 控制引脚 3 和 11

一些复杂的库（如舵机库Servo.h、红外接收库IRremote.h）为了实现其功能，需要接管（Reconfigure） 整个定时器。

  • 舵机库 (Servo.h) 与 引脚9和10 (Timer1)：
    • 舵机需要非常精确的50Hz PWM信号。
    • 为了方便和保证精度，Servo.h库默认会接管16位的Timer1，将其强制设置为产生50Hz的PWM模式。

原理

模拟引脚的实现依赖于一个叫做 ADC (Analog-to-Digital Converter，模数转换器) 的硬件模块。它的任务是将一个连续的模拟电压信号，转换成一个离散的数字数值，以便MCU的数字核心能够读取和处理

当你执行 analogRead(A0)时，底层硬件悄然完成以下操作：

  1. MCU内部的多路选择器将A0引脚连接到ADC模块。
  2. ADC启动一次转换。
  3. 采样保持电路捕获A0引脚上的瞬时电压。
  4. ADC进行量化，将电压值转换为0-1023之间的数字。
  5. 转换完成，结果被存入数据寄存器。
  6. analogRead()函数从这个寄存器中取出这个值并返回给你的程序。

详细如下:

  1. 采样 (Sampling)

    ADC并不是持续不断地测量电压，而是以一个固定的时间间隔（由采样频率决定），快速地“瞥一眼”输入引脚上的电压值，并把这个瞬间的电压值捕获下来

采样率 (Sample Rate)。根据奈奎斯特采样定理，采样频率必须至少是输入信号最高频率的2倍，才能无失真地还原原始信号。但对于大多数Arduino测量的传感器（如缓慢变化的温度、光线），默认的采样率已经足够

  1. 量化 (Quantization)

    捕获到的模拟电压值可以是某个范围内的任意值（如3.1415926… V）。但ADC需要把它“归类”到一个有限的、离散的等级中去。这就是分辨率的概念

    分辨率 (Resolution)，通常用位数 (bits) 表示。

    • Arduino UNO的ADC是 10位 的。这意味着它可以将 0V至5V 的参考电压（Vref）划分为 2¹⁰ = 1024 个离散的等级（从0到1023）。
    • 每个等级代表的电压差是： 5V / 1024 ≈ 0.00488V (4.88mV)

  2. 编码 (Encoding)

    将量化后得到的等级数值，转换成一个二进制数字，并存储到MCU的寄存器中，等待程序通过 analogRead()函数来读取

    最终在代码中得到的 int sensorValue = analogRead(A0);里的 sensorValue，就是这个编码后的数字值（0-1023）

相关参数

  1. 分辨率 (Resolution)

    • 定义：ADC能区分的最小电压变化能力，位数越高，分辨率越高。
    • 举例：
      • 10位 (Arduino UNO)：范围 0-1023，最小电压步进 ~4.88mV。
      • 12位 (STM32, ESP32)：范围 0-4095，最小电压步进 ~1.22mV（假设5V参考电压）。精度是10位的4倍！
    • 影响：分辨率决定测量的精细度，高分辨率ADC能探测到更微弱的电压变化。

  2. 参考电压 (Reference Voltage, Vref)

    • 定义：ADC用于比较和量化的基准电压，通常是MCU的工作电压（如Arduino UNO为5V）。
    • 重要性：测量小电压范围（如0-1V）时，若使用5V参考电压，精度会大幅浪费。
    • 解决方案：许多MCU允许改变参考电压，使用analogReference()函数可改变Arduino的参考电压，提高精度。

  3. 采样速度 (Conversion Speed)

    • 定义：ADC完成一次采样-量化-编码过程所需时间。
    • 影响：决定能测量多快变化的信号，捕获音频信号（20kHz）需高采样速度（>40kHz），而测量温度等缓慢变化的数据，低速ADC也足够。

螺丝

螺丝的关键组成部分

  1. 头部：提供驱动和承压面。
  2. 驱动方式：头部上的凹槽，用于传递扭矩。

  4. 杆部/光杆：头部以下、螺纹以上的部分。
  5. 材质与表面处理：决定其强度、耐腐蚀性和外观。

分类

  • 按驱动方式分类(用什么工具拧)

    这是最直观的分类方式，决定了你需要使用什么螺丝刀或扳手

    • 外六角/六角头螺丝：用扳手在螺丝头部外侧操作，可施加较大扭矩，常用于机械、设备等重型结构。

    • 十字槽/菲利普斯螺丝：最常见的螺丝之一，带有自对中功能，比一字槽更不易打滑，但扭矩过大时仍会“跳齿”。
    • 梅花槽/星形槽螺丝：能传递比十字槽更大的扭矩，更防打滑，常见于电子产品、电器、汽车。
    • 方头螺丝：防打滑性能好，但不如十字槽和梅花槽普及。
    • 复合槽螺丝：结合了两种槽型，如一字/十字复合，增加工具兼容性。
    • 特殊安全槽螺丝：如三角槽、蛇形槽、单向槽等，主要用于公共设施，需要专用工具，防止被随意拆卸。

  • 按头部形状分类

    头部形状影响安装后的外观、承压能力和沉入方式

    1. 盘头/平圆头：头部凸出，承压面较大。

      在绝大多数情况下，人们说的“圆头”就是指盘头

    2. 沉头/平头：头部为圆锥形，可完全沉入工件表面，使表面平整。

    3. 半沉头/椭圆头：头部顶端为弧形，部分沉入工件，比沉头更美观。

    4. 圆柱头：头部为圆柱形，需配合垫片使用，承压面大。

    5. 球面圆柱头/蘑菇头：外观圆润，常用于有装饰要求的场合。

      这其实是更标准、更圆的“圆头”。它的侧面看是一个完美的半圆形，像蘑菇的帽子。它比盘头更圆润。所以它也属于“圆头”的范畴

    6. 六角头：见“驱动方式”分类，通常不沉孔，用扳手操作。

    7. 内六角圆柱头：结合了内六角驱动和圆柱头形状，非常常见。

    8. 带垫圈头：头部自带垫圈，增大承压面，防松，一步到位。

  • 按牙型/螺纹类型分类（核心功能分类）

    这是最专业、最重要的分类方式，直接决定螺丝的用途

    1. 机械螺丝

      • 特点：螺纹为标准公制粗牙或细牙，通常配合预制好的内螺纹（如螺母、螺纹孔） 使用。
      • 牙型：一般为三角牙，夹角60度。
      • 应用：通孔连接，需要与螺母配合，或拧入已攻好丝的螺纹孔中。

    2. 自攻螺丝

      • 特点：螺纹间距较宽，牙顶更尖锐，能够在被连接材料（如塑料、木材、薄金属板）上自己攻出内螺纹，无需预制螺纹孔或螺母。
      • 细分：
        • AB牙/尖尾自攻丝：用于塑料、木材、铝材等软质材料。
        • BT牙/平尾自攻丝：用于钢结构、较厚的金属板，硬度更高。
        • 高低牙自攻丝：高低交替的螺纹，用于塑料，拧入阻力小，锁紧力强。
        • 三角牙自攻丝：螺纹截面呈三角形，用于塑料，防松效果好。
        • 喇叭孔/钻尾自攻丝：尾部带钻头，可在薄钢板、铝型材上自钻、自攻，无需预打孔。

    3. 木螺丝

      • 特点：可视为一种特殊的自攻螺丝，螺纹深、间距大、牙顶尖锐，专门为木材设计。杆部通常为锥形（从尖到头逐渐变粗）。
      • 应用：木材与木材之间的连接

    4. 螺栓

      与螺丝的区分：通常指配合螺母使用，用于通孔连接的紧固件，头部多为六角头，用于需要较大预紧力的重要连接。有时与“螺丝”概念有交叉

    5. 螺杆

      特点：全杆都是螺纹，没有特定的头部，通常配合两个螺母使用，用于定位、传动或调节

  • 按材质分类

    • 碳钢：最常用，强度高。按强度分为4.8级、8.8级、10.9级、12.9级等（数字越大强度越高）。
    • 不锈钢：防锈耐腐蚀。常见牌号有SUS304（A2） 和SUS316（A4，耐海水腐蚀更强）。
    • 铜：导电性好，耐腐蚀，常用于电工、船舶。
    • 铝合金：重量轻，有一定强度，耐腐蚀，但硬度较低。
    • 钛合金：强度极高，重量轻，耐腐蚀，用于航空航天、医疗等高价值领域。
    • 塑料：如尼龙，绝缘、防腐蚀、重量轻，用于轻负载或腐蚀环境。

  • 按表面处理分类

    主要为了防锈、美观

    • 镀锌：最常见，有白锌（耐腐蚀性一般）、蓝白锌、彩锌（耐腐蚀性较好）。
    • 镀镍：银白色，美观，耐腐蚀性好。
    • 发黑/发蓝：表面为黑色氧化膜，防锈能力一般，主要用于美观和减反光。
    • 达克罗/锌基铬酸盐：无氢脆，耐腐蚀性极佳，环保。
    • 热浸镀锌：镀层厚，耐腐蚀性强，但外观粗糙，多用于户外工程。
    • 不锈钢本色：无需处理，本身具有耐腐蚀性。

  • 按用途/功能分类

    • 紧固螺丝：最普通的螺丝，主要功能是连接和紧固。
    • 定位螺丝：通常无头，如止付螺丝/机米螺丝，用于固定零件之间的相对位置（如固定轴套）。
    • 膨胀螺丝：用于混凝土、砖墙等基体，通过膨胀套筒产生膨胀力来实现锚固。
    • 吊环螺丝：头部为环形，用于起吊、悬挂。
    • 止退螺丝/防松螺丝：具有防松功能，如尼龙圈螺丝、点胶螺丝、双螺母等。
    • 焊接螺丝：一端可焊在金属件上，另一面形成螺纹孔。
    • 家具螺丝：如三合一连接件、偏心轮螺丝等，用于板式家具的组装。

在实际应用中，一颗螺丝的描述通常是以上分类的组合。例如：

“一颗M6x20 不锈钢A2-70 内六角圆柱头 机械螺丝”

  • 尺寸： M6x20（公制，直径6mm，长度20mm）
  • 材质： 不锈钢A2-70（304不锈钢，强度700MPa）
  • 驱动方式： 内六角
  • 头型： 圆柱头
  • 牙型： 机械螺丝（需配螺母或螺纹孔）

线材

自制杜邦线

自制杜邦线其实是由三部分组成的：电线、金属端子、塑料胶壳。

舵机线的三色线有严格标准，接错可能烧毁舵机或控制器！务必遵循：

  • 棕色/黑色：GND（地线）
  • 红色：VCC（电源正极，+5V）
  • 橙色/黄色/白色：Signal（信号线）

自制时，必须从头到尾保持这个顺序一致！

p.s. 舵机三色线就是三根并排的杜邦线。

采购建议:

电线选择

优先搜索“22AWG 三色硅胶线”或“舵机延长线 硅胶线”

特性 推荐选择 为什么重要？
线规（粗细） 推荐 AWG22 或 AWG24 舵机工作需要一定电流，线太细（如AWG28）电阻大，会导致压降，舵机没力气或 jitter（抖动）。AWG22（约0.3mm²）或AWG24（约0.2mm²）是舵机线的标准规格。
线芯结构 多股绞合铜线 非常柔软，耐弯折，适合需要活动的接线。绝对不要选单芯硬线，一弯就断。
线皮材质 硅胶线（首选）或PVC软线 硅胶线极其柔软，耐高温，是专业模型的首选。普通PVC软线也可以，但要选手感软的。
颜色 三色并行线 或 红/黑/黄三色散线 并行线方便整理，但硅胶材质的并行线较少。买三色散线自己组合也可以，务必保证颜色对应，以免接错。

端子和胶壳的选择（确保物理兼容）

杜邦线端子是标准件，但也要注意细节。

部件 关键点 注意事项
金属端子 选择 杜邦线专用端子，通常是 PH2.0 系列。 一定要匹配线径！购买时确认端子适用于AWG22-24或AWG24-26的电线。
塑料胶壳 选择标准的 2.54mm间距（0.1英寸） 的杜邦线胶壳。 买一排多个的自己用钳子压断，比单买几个更划算。注意公母头：线对板一般是公头插针，母头胶壳。

杜邦接头（公头/母头）：

  • 这是什么： 就是插在舵机、PCB板上的那个小小的塑料头，里面是金属针。杜邦线需要两端都有接头。
    • 公头：像针一样，用于插入主控板（如PCA9685）上的插针。
    • 母头：像插座一样，用于插入舵机（舵机引线上自带的是公头）。
  • 如何购买： 搜索“杜邦头”或“杜邦接头”，通常会按“对”（一个塑料壳+一个金属端子）来卖，需要购买一大包，包含公头和母头

2.54mm是标准间距,也是最常见的一种

  1. 杜邦母头：用于连接舵机（因为舵机自带的线是公头）和主控板上的排针（主控板上的针是公的）。
  2. 杜邦公头：用于连接PCA9685舵机控制板等扩展板（因为这些板子上的接口也是母座）

必备工具（决定成败的关键）

工具 必要性 说明
专业压线钳 极高 这是最值得投资的工具。推荐购买杜邦线专用压线钳，一个钳口同时压接端子的导线部和线皮部，成功率高，连接牢固。绝对不要用普通钳子去捏，很容易接触不良或压断线芯。
剥线钳 高 能精准剥去线皮而不伤铜丝。
镊子 中 方便将小端子放入压线钳和胶壳。

压线钳的选择

  • 选择 0.08~1mm² 的精密压线钳，如果**：你的工作核心是电子制作，例如焊接你的机械臂时需要用到的杜邦线、舵机连接线、电路板上的飞线等。这款钳子的模具尺寸更小，能精准地压接纤细端子的金属部分，确保压接牢固不松动，这是制作可靠连接的关键**
  • 选择 0.25~1.5mm² 的通用压线钳，如果**：你经常需要处理家用电灯、插座、家用电器维修中遇到的普通电源线。它的压接范围更偏向日常电工标准，但对于极细的电子线缆可能无法保证压接质量。

可调节电源

  • 30V 10A可调节可满足绝大多数需求

  • 纹波噪声：线性电源≤5mVrms，开关电源尽量≤10mVrms；嵌入式开发优先线性，减少MCU/传感器干扰。

    警惕虚标纹波（如标“≤5mV”却不注明rms或p-p）

  • 线性电源

    线性电源是按电源的电路拓扑结构划分的，核心是通过线性调整管（如功率三极管）的线性压降来稳定输出，特点是纹波极低、噪声小，适合嵌入式精密调试。

  • 调节与模式：双旋钮/编码器更易精调；必须支持CV/CC自动切换，方便设定限流保护负载

    CV是恒压,CC是恒流
    自动切换逻辑：给电源设好目标电压（比如12V）和限流值（比如2A）。如果负载需要的电流≤2A，电源保持12V不变，工作在CV模式；如果负载突然短路，电流要飙升到10A，电源会把电流限制在2A，自动切到CC模式，避免烧坏电源和设备。

  • 四位显示: 注意要确认是0.01V/1mA步进，而非仅显示位数。

配网开发

AP 热点 + Web 配置页面

工作原理

  1. 启动检测：ESP32 启动时，先尝试连接之前保存的 WiFi。
  2. 开启热点：如果连接失败（或第一次开机，或用户长按物理按键），ESP32 会进入 AP (Access Point) 模式，发射一个特定的 WiFi 热点（例如：ESP32-Config-AP）。
  3. 手机连接：用户用手机连接该热点。
  4. 强制门禁（Captive Portal）：连接后，手机会自动弹出一个网页配置页面（类似连接酒店/机场 WiFi 时的登录页），或者用户在浏览器中手动输入 192.168.4.1。
  5. 提交配置：网页展示扫描到的 WiFi 列表，用户选择自己的 WiFi 并输入密码，点击“保存”。
  6. 连接并保存：ESP32 收到密码后保存到 Flash（NVS/Preferences），重启并切换到 STA 模式 连接用户的 WiFi。

WiFi 智能配网与 WiFiManager

当物联网设备（如 ESP32）出厂时，它是没有用户家里 WiFi 账号密码的。为了联网，我们需要“配网”。

强制门户配网 (Captive Portal) 的过程

  1. AP 热点模式：ESP32 启动自己内部的 WiFi 发射模块，建立一个没有密码的局域网热点（例如 ESP32_WiFi_Config）。
  2. DNS 重定向：当用户的手机连接这个热点时，手机会自动尝试去访问互联网（如测试苹果或安卓的连通性域名）。ESP32 内部运行一个 DNS 服务器，将所有域名解析请求全部拦截并强行指向自己的 IP（192.168.4.1）。
  3. 自动弹窗：手机发现访问的域名返回的不是标准握手，而是 ESP32 网页服务器（WebServer）提供的网页，手机系统就会自动弹出这个“配网网页”。
  4. 保存与重启：用户输入密码后，ESP32 将其写入内部的 NVS (Non-volatile storage, 非易失性存储区)。重启后，ESP32 就可以从 Flash 中读取密码，直接连接用户的路由器。

PN结

单向导电性的原理: 内部有一个PN结

科学家发现,利用掺杂工艺,将多种不同的物质掺杂在硅片中,在同一个硅片上会形成两种不同的部分

  • 本征硅（纯硅）导电能力很弱；
  • 掺入三价杂质（硼、铝等），形成P 型半导体，主导导电的是空穴；
  • 掺入五价杂质（磷、砷等），形成N 型半导体，主导导电的是自由电子。

正极的叫P型半导体,负极的叫N型半导体,这两个部分接触的这个平面会形成一个特殊的现象,电子只能从合金N流向合金P,反之则不能

把 P 型硅和 N 型硅结合在一起，接触面会形成PN 结，这就是单向导电的根源

PN 结区域会自发形成耗尽层和内建电场：

  • 加正向电压（P 接正、N 接负）：内建电场被抵消，载流子顺利穿越结区，电路导通；
  • 加反向电压（P 接负、N 接正）：内建电场被加强，载流子无法穿越，电路近似截止。

PN结原理参考

硅是晶体，原子被共价键牢牢固定在晶格中，晶格本身的离子不能移动，只有两种 “可自由跑动” 的导电粒子（载流子）：电子（负电）、空穴（等效正电）。

P、N 半导体结合 → 载流子浓度差引发扩散 → 界面附近可移动载流子跑光，只剩固定离子 → 形成耗尽层 → 两侧正负固定电荷分层 → 产生内建电场 → 电场阻碍扩散，最终双向运动平衡

宏观上不再有载流子的净移动，耗尽层的宽度、内建电场的强度就彻底稳定下来了。

这就是 PN 结自然形成后的热平衡状态

  • 正向偏置（P 接电源正极，N 接负极）

    外接电场抵消了内部的内建电场，“反向风” 消失，载流子可以顺利穿过耗尽层 → 导通；

  • 反向偏置（P 接电源负极，N 接正极）

    外接电场加强了内部的内建电场，“反向风” 更大，耗尽层变宽，载流子完全无法穿过 → 截止。

这是半导体特有的能带、载流子输运效应

二极管 -> 三极管 -> 集成电路芯片 -> 单片机 -> 电脑cpu ->显卡 -> AI大模型

正是因为有PN结,👆🏻才有了实现的可能

  • 集电极 Collector (c) 轻掺杂，掺杂浓度低；结面积远大于发射结（专门用来收集载流子）；反向耐压很高（几十～上百伏）
  • 基极 Base (b)
  • 发射极 Emitter (e) 重掺杂，掺杂浓度极高，作用是大量发射电子；结面积小；反向耐压极低（一般仅 3～8V）。

虽然三极管外观上 C、E 引脚位置相近、都是同类型半导体区（如 NPN 两侧都是 N 型），但内部掺杂浓度、结面积、耐压、设计功能完全不对称；强行互换后管子性能大幅劣化、参数剧变，电路会工作异常，甚至直接击穿损坏。

三极管的缺陷

晶体管进化史: PN结 -> 二极管 -> 三极管 -> MOS管

随着市场上对集成电路的密度要求越来越高,基于PN结的三极管无法满足需求

三极管缺陷盘点

  • 通过电流有限,损耗缺陷与体积缺陷
  • 属于电流器件,无法电压控制
  • 生产工艺受限,无法高集成化

MOS管

1947年 贝尔实验室
科学家威廉.肖克利与同事们发明了
双极型晶体管（三极管）和场效应管 （FET）
因此，威廉.肖克利获得1956年诺贝尔物理奖

JFET 结型场效应管

利用P极上负电压产生绝缘区域
是属于电压生成的“电场效应“
由这种电场效应来控制的半导体结构就叫**“场效应管”**

但JFET场效应管还有致命缺陷

由于被控制的大电流只在同一个N型半导体内通过,电流不需要穿过PN结,所以工作稳定、干扰噪声小.因此非常适用于放大电路

但如果用于开关电路,由于使用夹紧的方式断开电流,夹不紧的时候会有小电流漏出

因此还需要改变半导体结构

  1. 最好初始无电压断开
  2. 加电压后导通

1959年 结构2.0改造成功,增强型MOSFET管诞生了

增强型MOSFET

简称MOS管

D和S不能调换使用,因为其中会有寄生二极管,有一些原理图中会画出这个寄生二极管

寄生二极管还有一个妙用,当我们使用MOS管控制电机,电磁体之类的电感器件,寄生二极管还能反向放电,保护MOS管不被反向电流损坏

项目完整目录

ESP32 芯片和 Vue 3 前端网页被看作是两个独立的项目：

  1. 后端（ESP32 C++）
    • 角色：数据源与硬件控制器 (API Server)。
    • 任务：挂载文件系统、监听网络、读取底板按键 GPIO、控制 LED。它不负责画网页，只负责通过 /api/status 接口输出 JSON 格式的数据（如 {"pressCount": 5, "wifi": true}），或者接收网页发来的清零请求。
  2. 前端（Vue 3 单页应用）
    • 角色：用户交互界面 (User Interface)。
    • 任务：独立开发、独立排版。通过 fetch() 函数（类似于浏览器中的 AJAX 请求）定时向底板的 API 发送请求，获取最新的状态数据，并使用 Vue 的“响应式”机制动态刷新页面，同时向底板发送控制指令。

核心机制

文件系统 (LittleFS) 与 Gzip 压缩

  • 什么是 LittleFS？ LittleFS 是专为微控制器设计的轻量级安全文件系统。我们可以把 ESP32 的 Flash 闪存（通常有 4MB）划分出一部分（比如 1MB）当成“U盘”来用。
  • Gzip 压缩： 现代前端打包出的 index.html、js 和 css 虽然经过压缩，但体积依然不小。我们的构建管道会自动使用 Gzip 最大压缩算法 (Level 9)，将它们全部压缩成 .gz 格式（如 index.html.gz）。
  • 浏览器自动解密： 当用户访问开发板的 IP 时，ESP32 会直接把这个压缩包 index.html.gz 发给浏览器，并附带响应头 Content-Encoding: gzip。现代浏览器（Chrome/Safari等）收到后，会在底层自动解压并渲染，ESP32 甚至不需要多花 1 微秒的 CPU 去解压它。这既节省了空间，又极速提升了网页加载速度。
