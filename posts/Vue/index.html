Vue相关知识

vue

一个JavaScript框架,目的是将html和JavaScript连起来,让网页能动起来

下面只介绍了基本用法,详情参考官方文档

一个Vue页面分为三个部分

  • script标签: 写js的地方
  • template标签: 写HTML的地方
  • style标签: 写css的地方

vue中支持在html中使用变量通过{{变量名}}方式

最基础的一些使用方法：

  • 直接将数据渲染到dom
  • v-bind:attribute名=”Vue的键”,将属性绑定到Vue的键
  • v-if=”Vue的键”,将Vue的键（bool值）绑定元素是否显示
  • v-for=”xxx in xxxs”,将Vue的键xxxs(数组)，数组全部渲染出来
  • v-on:click=”函数名”,绑定函数到事件，函数直接在Vue实例对象中，可以直接使用那些数据
  • v-model=”Vue的键”，双向绑定输入和变量

vue中创建响应式变量

创建的响应式变量,在template的""中可直接使用str,在标签之间的文本中通过{{str}}来使用;而在script中,通过str.value来使用

{{}}中是可以执行运算公式的

在Vue中，响应式变量是指当数据发生变化时，页面会自动更新以反映这些变化的特性。Vue通过使用数据绑定和依赖追踪来实现响应式变量。

当你在Vue实例中声明一个变量时，Vue会将这个变量转换为响应式变量。这意味着当这个变量的值发生变化时，与之相关联的视图会自动更新以反映这个变化。

Vue实现响应式变量的原理是通过使用Object.defineProperty()方法来劫持数据的访问和修改，从而能够在数据发生变化时通知相关的视图进行更新。这种机制使得开发者无需手动操作DOM，而是专注于数据的处理和逻辑，让Vue框架来处理视图的更新。

当使用ref属性引用一个响应式变量时，当这个响应式变量发生变化时，通过ref引用的元素也会相应地更新。这样，ref属性可以帮助我们在需要直接访问DOM元素或组件实例时，保持视图与数据的同步更新。

插值表达式

基本用法

  • 语法：{{ 数据/表达式 }}
  • 作用：将数据渲染到页面

支持内容

  • 变量：{{ msg }}
  • 表达式：{{ num + 1 }}、{{ msg.split('') }}
  • 三元运算：{{ isOk ? '是' : '否' }}
  • 计算属性：{{ fullName }}

不支持

  • 语句：if、for、var a = 1
  • 分支循环，只能用指令

特点

  • 响应式：数据变，视图自动更新
  • 只会渲染文本，HTML 会被转义

一次性插值

{{* msg }}（数据变了也不更新，极少用）

事件绑定

监听 DOM 事件（点击、输入、滚动等），触发对应的 JS 函数

指令：

  • 完整写法：v-on:事件名
  • 简写（推荐）：@事件名

支持所有原生 DOM 事件：click / keyup / keydown / input / submit / scroll 等

绑定的事件是否带括号的区别

  • 绑定方法名（无括号）

    • 直接传递函数引用，Vue 自动调用
    • 自动传递原生事件对象 $event
    • 无需传参时首选，简洁无错

  • 内联调用（带括号）

    • 手动执行函数
    • 不会自动传事件对象，需要手动传 $event
    • 用于自定义传参场景

数据绑定

变量绑定

  1. :value="msg" → 数据 → 视图（单向绑定）

  • : 是 v-bind 的简写，给标签属性绑定 Vue 数据

  • 作用：把 Vue 里的变量 msg，赋值给输入框的 value 属性

  • 效果：

    数据变，输入框显示的内容自动变

    • 例：msg = "你好" → 输入框自动显示「你好」
  1. @input="msg = $event.target.value" → 视图 → 数据（事件监听）
  • @ 是 v-on 的简写，监听 DOM 原生事件
  • input 事件：用户每输入一个字、粘贴内容时，都会触发
  • 后半段是事件的执行逻辑：
    • $event：Vue 提供的原生事件对象
    • $event.target：触发事件的元素（就是这个输入框）
    • $event.target.value：输入框里当前输入的最新文字
    • msg = ...：把输入的新值，重新赋值给 Vue 变量 msg
  • 效果：用户输入内容，Vue 数据自动更新

v-model是上面的语法糖!

value变量与input控件双向绑定成功

  • 如果需要在script中使用响应式变量的值需要使用:value.value,前一个value是绑定的响应式变量,后一个 value才真正取到变量
  • 如果需要再template中使用响应式变量的值,只需要直接使用变量名字
  1. v-bind用于动态绑定数据到HTML元素的属性上，实现数据的单向绑定。通过v-bind，你可以将Vue实例中的数据动态地绑定到HTML元素的属性上。
  2. v-model用于实现表单元素和Vue实例数据的双向绑定。当表单元素的值发生变化时，Vue实例中的数据也会相应地更新；反之，当Vue实例中的数据发生变化时，表单元素的值也会随之更新。

条件渲染

条件渲染根据条件真假，动态控制元素的显示 / 隐藏 / 创建销毁，核心指令：v-if、v-else-if、v-else、v-show，是 Vue 最基础、最常用的指令之一

特性 v-if v-show
渲染原理 条件假 → 移除 DOM 条件假 → 隐藏（display:none）
创建开销 低（不创建） 高（始终创建）
切换开销 高（频繁创建 / 销毁） 低（仅切换样式）
支持模板 支持 <template> 不支持
支持 else 支持 v-else/v-else-if 不支持
适用场景 切换不频繁、权限控制 频繁切换（tab、折叠面板）

注意事项

  • Vue3 中 v-if 优先级高于 v-for，绝对不要同标签使用

computed计算属性

computed 是 Vue 提供的计算属性，用来根据依赖的数据动态计算出新值。

  • 基于依赖缓存（依赖不变，多次调用只计算一次）

    依赖的 data /ref/reactive 变化时，计算属性自动更新，视图自动刷新

  • 像属性一样使用（不用加括号）

  • 必须有返回值

  • 比 methods 性能更好

    • 依赖数据没变化 → 直接拿缓存，不重新计算
    • 依赖数据变化 → 自动重新计算

默认computed只有getter(只读),可以手动写setter实现修改

常用场景

  1. 拼接字符串（姓名、地址）
  2. 数字计算（总价、折扣、倍数）
  3. 列表过滤 / 排序（搜索功能）
  4. 表单数据处理
  5. 根据状态动态计算样式 / 类名
特性 computed watch methods
缓存 有 无 无
使用方式 当作属性使用 监听数据变化 当作方法调用
依赖 多依赖 单 / 多依赖 无依赖
返回值 必须 return 不用 return 可 return 可不 return
适用场景 计算值、模板使用 异步、复杂逻辑 事件处理、普通方法

事件修饰符

事件修饰符是 Vue 为 DOM 事件 提供的语法糖，用于快速处理事件的默认行为、冒泡、捕获、触发次数等场景，无需手动编写 event.preventDefault() / event.stopPropagation() 等原生代码，直接拼接在 v-on / @ 事件后即可使用。

Pinia

Pinia 是 Vue3 官方推荐的全局状态管理库，是 Vuex 的继任者，专门用来解决 Vue 项目中全局数据共享的问题。

简单说：它是一个全局数据仓库，项目里所有组件、页面，都可以从这里读数据、改数据。

官方文档

安装 npm install pinia

核心作用

  • 跨任意组件共享数据（父子、兄弟、跨级、跨页面都可以）
  • 统一管理全局状态（用户信息、登录状态、购物车、主题等）
  • 数据响应式同步：一处修改，所有使用的组件自动更新
  • 管理异步数据（接口请求、数据缓存）

核心特点：

  • 极简 API，移除了 Vuex 中的 Mutations，直接通过 Actions 处理同步 / 异步操作
  • 完整的 TypeScript 支持，开箱即用，无需额外配置
  • 体积小巧（约 1.5KB），性能优于 Vuex
  • 模块化设计，支持多 Store，无单一状态树限制
  • 兼容 Vue DevTools，支持时间旅行调试

四大核心概念

Store仓库

Store 是存放状态数据、计算属性、业务方法的独立容器，你可以理解为全局数据仓库。

一个项目可以创建多个独立 Store，按业务模块拆分（用户、购物车、主题、计数器等），彼此互不干扰。

核心作用

  1. 统一管理某一类业务的全局数据与逻辑；
  2. 脱离组件独立存在，任意组件都可以导入并使用；
  3. 实现跨组件、跨页面的数据共享。

关键规则

  • 创建 Store 必须使用 Pinia 内置方法：defineStore(唯一ID, 配置)
  • 第一个参数是 Store 唯一标识 ID（字符串），全局不能重复，用于调试、持久化等；
  • 推荐目录：项目 src/stores/ 文件夹，一个业务对应一个文件（如 user.js、cart.js）。
State状态(原始数据源)

State 是 Store 中原始、基础的响应式数据，相当于组件里的 data / ref/reactive，是整个状态管理的数据源

核心作用: 存储项目中需要全局共享的原始数据：用户信息、Token、购物车列表、计数器数值、主题配置等

修改 State 的 4 种常用方式

注意点

  1. 数据天生响应式，一处修改，所有使用该数据的组件自动更新视图；
  2. 不要在组件中写大量业务逻辑修改 State，复杂逻辑统一放到 Actions；
  3. 嵌套对象修改无需额外处理，原生响应式生效。
Getters 计算属性（派生数据）

Getters 是基于 State 派生出来的计算数据，完全等价于 Vue 组件中的 computed 计算属性

核心作用

  1. 对原始 State 做二次计算、格式化、筛选；
  2. 自带缓存：只有依赖的 State 发生变化时，才会重新计算；
  3. 复用通用计算逻辑，避免多个组件重复写相同代码。

使用方式

特点

  • 只读：Getters 仅用于取值计算，不能在这里修改 State；
  • 缓存特性：普通 getter 会缓存，传参 getter 无缓存；
  • 常用场景：总价计算、数据筛选、文本格式化、权限判断。
Actions 动作方法（业务逻辑层）

Actions 是 Store 中封装的函数 / 方法，等价于组件中的 methods。

它是 Pinia 中修改 State、处理业务逻辑、请求接口的标准入口

核心作用

  1. 封装同步 / 异步逻辑，统一修改 State；
  2. 处理网络请求（接口调用）、定时器、表单校验等复杂业务；
  3. 支持跨 Store 调用，实现多仓库联动。

核心优势（对比 Vuex）

✅ 不再区分 mutations 和 actions，同步、异步代码都可以写在 Actions 里，大幅简化代码；

✅ 支持传参、async/await 异步语法；

✅ 内部可直接通过 this 访问 State / Getters。

特点 & 适用场景

  1. 推荐规范：复杂数据修改、异步请求、业务逻辑，一律写在 Actions 中；
  2. 可传参、可异步、可跨仓库调用；
  3. 适用场景：登录 / 登出、提交表单、接口请求、批量数据操作。

项目结构规范与最佳实践

针对中大型 / 超大型 Vue3 工程（后台管理、电商、中台、多业务线系统），Pinia 不能简单把所有仓库堆在一个文件夹里，必须依靠分层目录、单一职责拆分、统一编码规范、工程化约束来保障可维护性、协作效率与性能。

哪些数据进入Pinia

✅ 放入 Pinia（全局共享状态）

  • 跨页面 / 跨组件复用的数据：用户登录态、Token、权限、角色；
  • 系统全局配置：主题、语言、布局、菜单、标签页；
  • 业务全局数据：购物车、全局字典、公共枚举、常驻筛选条件；
  • 需要持久化的数据：用户偏好、登录凭证。

❌ 禁止放入 Pinia（页面 / 组件临时状态）

  • 仅当前组件 / 页面使用：弹窗显隐、开关、表单临时输入值、分页页码、局部搜索关键词；
  • 一次性接口返回数据：列表临时数据、弹窗详情（关闭页面即销毁）。

规范：临时状态直接用组件内 ref/reactive，Pinia 只托管全局、长期、多端共享的状态。

Store 单一职责原则
  • 一个 Store 只负责一个独立业务领域，拒绝 “巨型大仓库”（单个 Store 代码建议控制在 300 行内）；
  • 大业务域继续细分子模块，例如「订单」拆分为：订单基础、售后退款、订单统计、物流信息；
  • 公共能力与业务能力强制分层，全局公共 Store 独立抽离。
调用层级规则（单向依赖）
  1. 公共仓库（core/）：被所有业务仓库调用，不反向调用业务仓库；
  2. 业务仓库：可以调用 core 公共仓库；业务仓库之间尽量不互相调用；
  3. 业务模块之间需要交互：优先使用「路由参数、全局事件总线、后端接口」中转，迫不得已再调用其他业务 Store。

循环依赖避坑 不要在两个业务 Store 中互相导入，会造成运行报错。解决方案：把公共逻辑抽离到 utils 或提升为 core 公共仓库。

总结
  • 目录分层：核心公共库 + 业务域仓库 + 插件 + 工具，隔离清晰；
  • 拆分原则：按业务域拆分，单一职责，拒绝大仓库；
  • 状态边界：全局共享进 Pinia，页面临时状态放组件；
  • 编码统一：全项目一套写法、一套代码顺序、一套命名规则；
  • 通信约束：单向依赖，公共库被业务调用，业务之间尽量隔离；
  • 工程化兜底：ESLint、动态注册、精细持久化、单元测试保障质量。

真正的跳转操作

四种跳转方式

  • <router-link to="路径">：模板声明式静态跳转，生成 a 标签，保存历史记录，无 JS 逻辑，用于固定菜单跳转

  • <router-link :to="对象">：模板声明式带参跳转，支持 path+query、name+params 两种传参格式，页面导航传参首选

  • router.push()：JS 编程式跳转，追加浏览器历史栈，返回键可回退，项目日常业务跳转最常用，支持路径 / 命名路由 + 参数

  • router.replace()：JS 编程式替换跳转，覆盖当前历史记录，无法回退上一页，多用于登录页、404、表单提交后跳转

  • 补充：router.go(n)，前进后退，router.go(-1)返回上一页，不属于四大跳转，辅助页面回退

生命周期

生命周期：一个 Vue 组件从 创建 → 显示到页面 → 数据更新 → 销毁消失 的完整过程。

本质是一系列「钩子函数」，在特定阶段自动触发，让开发者能在恰当时机执行代码

核心价值：控制组件行为、管理资源（如 DOM 操作、数据请求、事件监听）、优化性能

阶段分类 生命周期钩子 执行时机 核心作用 高频使用场景
创建阶段 setup 语法块 组件初始化最早执行（DOM 未生成） 定义响应式数据、函数导入依赖、组合逻辑 初始化数据、业务逻辑封装
挂载阶段 onBeforeMount DOM 渲染到页面前 准备挂载前的配置 极少使用（基础业务忽略）
onMounted 组件渲染完成，真实 DOM 已生成 操作 DOM、请求接口、初始化插件 ✅ 发请求、图表 / 地图初始化、DOM 操作
更新阶段 onBeforeUpdate 数据变化，DOM 更新前 记录更新前的 DOM 状态 极少使用
onUpdated 数据变化，DOM 更新完成 操作更新后的 DOM 处理刷新后的 DOM 元素
卸载阶段 onBeforeUnmount 组件销毁前（DOM 仍存在） 清理资源、防止内存泄漏 ✅ 清除定时器、解绑全局事件、取消请求
onUnmounted 组件完全销毁（DOM 已移除） 收尾清理工作 极少使用

Capacitor

Capacitor 由 Ionic 团队开发，是老牌跨端工具 Cordova（PhoneGap）的官方现代继任者，采用 MIT 完全开源协议。它的定位是跨平台原生运行时—— 让标准 Web 应用能以原生 App 的形式运行在 iOS、Android 上，并统一调用手机原生能力

点击跳转到官网

它不是一个前端框架，而是一个「原生容器层」：你用 Vue/React/Svelte/ 原生 JS 写任何 Web 项目，都可以套上 Capacitor 直接打包成 App，几乎不用改业务代码

Capacitor 的核心架构是「系统 WebView + 标准化 JS - 原生桥接层 + 标准原生工程」：

  1. 把你的前端项目打包成静态 HTML/CSS/JS 资源，放入原生 App 的 WebView 中渲染；
  2. 通过统一的桥接层，让 JS 代码可以调用安卓 /iOS 的原生 API（相机、定位、文件等）；
  3. 生成标准的 Xcode（iOS）和 Android Studio（安卓）工程文件，原生代码完全可见、可编辑，不是黑盒基座。

这和 UniApp 的本质区别是：UniApp 会修改、阉割你的 Vue 代码以适配多端；Capacitor 完全不侵入你的前端代码，只在外部套壳。

核心能力

  • 全前端技术栈兼容：支持 Vue、React、Angular、Svelte 等所有主流框架，甚至原生 HTML 项目也能直接接入，零语法限制，DOM、BOM、CSS 全部标准可用。

  • 官方原生插件体系：官方维护了 30+ 常用原生能力插件，覆盖绝大多数业务场景，质量稳定且持续更新

    Capacitor：

    • 基础交互：相机、相册、剪贴板、分享、弹窗、操作菜单
    • 系统能力：定位、文件系统、网络状态、键盘控制、状态栏、震动反馈
    • 消息推送：本地通知、远程推送
    • 其他：浏览器、应用跳转、后台任务、设备信息

  • 丰富的生态兼容：社区有上千款第三方插件，同时向下兼容绝大多数 Cordova 旧插件，迁移成本极低Capacitor。

  • 原生工程完全可控：生成的 ios/android 目录是标准原生项目，可以直接用 Xcode/Android Studio 打开修改，支持自定义原生代码、接入第三方 SDK。

  • 调试体验拉满：支持 Chrome DevTools 远程调试 WebView，元素审查、断点调试、控制台、网络请求和调试普通 H5 完全一致。

  • 多端统一输出：同一套代码可生成 iOS App、Android App、标准 Web 站点、PWA 应用。

优势 劣势
完全不侵入前端代码，Web 生态 100% 兼容，所有 npm 库直接用 本质是 WebView 渲染，性能有天花板：超长列表、复杂逐帧动画、大型游戏易卡顿
工具链开放，不绑架编辑器，全程 VS Code 开发 超深度原生定制（如 AR、人脸识别、复杂硬件交互）仍需要原生开发基础
原生工程透明，没有云端打包黑盒，打包完全本地完成 官方无内置热更新方案，需用第三方服务（如 Capgo、Appflow）
调试排错效率远高于 UniApp，和 Web 开发体验一致 没有自带 UI 组件库，需自行搭配 Vant、NutUI 等移动端组件库
MIT 开源，无商业付费功能，没有功能阉割 —

性能差异

损耗的核心来源是底层的 WebView 渲染

  • 普通内容 / 表单页面：性能损耗极小，交互响应延迟与原生相差 50ms 以内，肉眼基本无感知，中高端机型可稳定 60fps。
  • 长列表 / 无限滚动：普通实现随条目增多会明显掉帧、黏滞；使用虚拟滚动优化后帧率接近原生，但快速滑动仍有白边占位，跟手性存在可感知差距。
  • 复杂动画与手势交互：性能损耗 30% 以上，易出现掉帧、卡顿、跟手性差，是 WebView 方案的核心短板。
  • 冷启动速度：比原生慢 25 倍，中高端机冷启动耗时约 500ms1.5s，用户可明显感知延迟。
  • 原生能力调用：桥接损耗极低，单次调用延迟约 2ms，用户完全无感知。
  • 内存占用：复杂场景下内存是原生的 2~3 倍，低端机多页面后更易被系统后台回收。

适用场景

✅ 完全够用，体验接近原生的场景

  • 工具类、资讯类、办公 OA、后台管理移动端
  • 普通电商、表单填写、用户中心类应用
  • 团队优先追求开发效率，性能只要 “不卡、能用” 即可
  • Web/H5 是主形态，App 只是配套输出

❌ 明显不够，不建议使用的场景

  • 信息流类无限滚动、短视频流（类似小红书、抖音）
  • 复杂手势交互、动画丰富的创意类 App
  • 直播、视频剪辑、AR/VR、大型游戏
  • 对启动速度、流畅度要求极高，对标一线大厂原生 App 的项目

Quasar

Quasar 是基于 Vue3 的一站式全栈跨端开发框架，同样 MIT 开源。它不止是一个 UI 组件库，而是包含了「组件库 + 工程化脚手架 + 多端构建能力 + 最佳实践」的完整解决方案，目标是让开发者只写一套 Vue 代码，就能输出到所有平台

点击跳转到官网

简单理解：它相当于把「Vue3 + Vite + 移动端 UI 库 + 路由 + 主题系统 + Capacitor 打包 + Electron 打包」全部帮你配置好了，开箱即用。

工作原理

Quasar 在上层提供统一的 Vue 组件和 API，底层通过不同的构建模式适配不同平台：

  • Web 端：直接编译为标准 SPA/SSR/SSG/PWA
  • 移动端 App：调用 Capacitor 或 Cordova 进行原生打包
  • 桌面端：调用 Electron 打包为 Windows/Mac/Linux 客户端
  • 浏览器扩展：编译为 Chrome/Firefox 扩展包

核心能力

  • 超全的内置 UI 组件：提供 200+ 高质量组件，采用 Material Design 设计风格，覆盖按钮、表单、导航、弹窗、列表、日期选择、上传、数据表格等几乎所有开发场景，无需额外引入第三方 UI 库。
  • 全平台构建能力：一条命令切换构建目标，支持 Web（SPA/SSR/SSG/PWA）、移动端 App（iOS/Android）、桌面端（Electron）、浏览器扩展，真正的「一套代码多端运行」Quasar Framework。
  • 工程化开箱即用：
    • 基于 Vite 构建，热更新速度快
    • 内置响应式断点系统，自动适配手机 / 平板 / 桌面
    • 内置主题定制、国际化（i18n）、图标库、权限管理
    • 内置性能优化、代码分割、资源压缩等最佳实践
  • Capacitor 深度集成：不用单独安装配置 Capacitor，Quasar CLI 直接封装了所有命令，开发、调试、打包 App 一条命令搞定。
优势 劣势
纯 Vue 技术栈，学习成本低，Vue 开发者上手几乎零门槛 默认 Material Design 风格，和国内主流 App 设计风格差异大，深度定制需要大量覆写样式
一站式解决方案，不用自己搭配技术栈，新项目搭建速度极快 App 底层依然是 WebView 渲染，性能上限和 Capacitor 一致
组件库质量高、覆盖全，业务开发效率高 组件库较重，全量引入会增大包体积（支持按需引入优化）
官方维护稳定，文档完善，社区活跃 生态广度不如纯 Vue + 第三方库的组合灵活

uni-app

拉胯,不是为了小程序都不建议学!

uni-app 是一个使用 Vue.js 语法进行跨平台应用开发的前端框架。它的核心价值在于，让你用熟悉的 Vue 技术栈，就能开发并发布到 iOS、Android、Web（H5）以及各种小程序等多个平台

特性维度 Vue.js uni-app
框架类型 用于构建用户界面的 渐进式 JavaScript 框架 基于 Vue.js 的 跨端应用框架
核心目标 专注于 Web 单页应用（SPA）开发 一套代码，多端发布
跨端支持 主要面向 Web 浏览器 支持发布到 iOS、Android、H5、微信/支付宝等各类小程序、快应用等十余个平台
标签规范 使用标准的 Web 标签（如 div, span） 使用小程序规范的标签（如 view, text），并映射为各端原生组件
路由管理 使用官方的 Vue Router 使用自带的路由框架，更接近小程序导航体验
性能特点 Web 应用性能 具备框架预载，页面加载更快；提供 uni-app x 等模式可编译为原生代码，实现更高性能
